# 競合記事分析ツール

競合サイトのブログ記事を自動収集・分析し、記事構成・キーワード・SEO要素を
**Excel / CSV レポート**として出力するPythonツールです。

**GitHub Actions で毎週自動実行**し、結果をダウンロードできます。

---

## GitHub Actions でのセットアップ手順

### 手順1：このリポジトリをフォーク（コピー）する

右上の **Fork** ボタンをクリックして、自分のアカウントにコピーします。

### 手順2：urls.txt に競合URLを書く

リポジトリ内の `urls.txt` を編集して、分析したい競合サイトのURLを1行ずつ書きます。

```
# 競合サイトのURLリスト（#から始まる行はコメント）
https://competitor-a.com/blog/article1
https://competitor-a.com/blog/article2
https://competitor-b.com/blog/seo-tips
```

GitHub上でファイルを直接編集できます（鉛筆アイコンをクリック）。

### 手順3：手動で試し実行する

1. リポジトリの **Actions** タブをクリック
2. 左側から「競合記事 定期分析」を選択
3. **「Run workflow」** ボタンをクリック
4. 少し待つと実行完了

### 手順4：結果をダウンロードする

1. Actions タブで実行済みのワークフローをクリック
2. 下部の **Artifacts** セクションに `competitor-analysis-xxx` が表示される
3. クリックしてZIPをダウンロード → 解凍するとExcelとCSVが入っています

---

## 自動実行のスケジュール

デフォルトは **毎週月曜日 午前9時（JST）** に自動実行されます。

変更したい場合は `.github/workflows/analyze.yml` の `cron` を編集します。

```yaml
schedule:
  - cron: "0 0 * * 1"   # 毎週月曜 AM 9:00 JST
```

| 実行したい頻度 | cron の値 |
|--------------|-----------|
| 毎週月曜 AM9時 | `0 0 * * 1` |
| 毎日 AM9時 | `0 0 * * *` |
| 毎月1日 AM9時 | `0 0 1 * *` |

---

## 出力されるExcelの内容

| シート名 | 内容 |
|---------|------|
| サマリー | ドメイン別の比較（平均文字数・SEOスコアなど） |
| 記事詳細 | 記事ごとの全指標（SEOスコアで色分け） |
| キーワード | 頻度・TF-IDF・共起ペアのランキング |
| 見出し一覧 | H1〜H4の全見出し（レベルで色分け） |

## 注意事項

- スクレイピングは `robots.txt` を尊重します
- 商用利用の際は対象サイトの利用規約を確認してください
